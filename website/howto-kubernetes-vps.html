How did you installed Kubernets on an Ubuntu VPS?

  1. Take a look at the official documentation:
    Kubernetes.io - Creating a single master cluster with kubeadm
  2. Upgrade installed packages:
    sudo apt update && apt upgrade
  3. Open Kubernetes API server port on the firewall:
    sudo ufw allow 6443/tcp #UFW example
  4. Install Docker:
    Check last supported Docker version for Kubernetes
    Add APT repository:
    sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" sudo apt update sudo apt install docker-ce=18.06.0~ce~3-0~ubuntu # Last compatible Version at 2018/11/19 sudo systemctl enable docker
  5. Install kubeadm, kubellet and kubectl:
    Kubernetes.io - Install kubeadm
    sudo apt install -y apt-transport-https curl sudo curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add - sudo apt update && sudo apt install -y kubelet kubeadm kubectl
  6. Prevent automatic package upgrades for the stack: sudo apt-mark hold docker-ce kubelet kubeadm kubectl
  7. Kubernetes needs swap to be disabled: sudo swapoff -a Also, check and disable '/etc/fstab' entries for swap
  8. Create the k8s cluster:
    I have chosen calico networking addon
    . So I needed to run 'init' parameter with calico required option: sudo kubeadm init --pod-network-cidr=192.168.0.0/16 If everything is ok, look for this message at the end of the output: Your Kubernetes master has initialized successfully! Follow the instructions to use your kubeconfig file for remote connections to the cluster
  9. Install CNI:
    Run these commands from your local kubectl context pointing to the new cluster.
    First, rbac security: kubectl apply -f https://docs.projectcalico.org/v3.3/getting-started/kubernetes/installation/hosted/rbac-kdd.yaml Finally, networking: kubectl apply -f https://docs.projectcalico.org/v3.3/getting-started/kubernetes/installation/hosted/kubernetes-datastore/calico-networking/1.7/calico.yaml
  10. Look for ready master:
    kubectl get nodes 
    # Output
NAME                     STATUS    ROLES     AGE       VERSION
vmi*.contaboserver.net   Ready     master    75m       v*.*.*
  11. As we are running a single node cluster, it's necessary to untaint the master node to be able to schedule pods: kubectl taint nodes --all node-role.kubernetes.io/master-
